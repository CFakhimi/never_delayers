  1. The Relational Schema

User(username, password)

  • username → password
  • Key = username

Route(airline, startAirport, destinationAirport)

  • Key = airline, startAirport, destinationAirport

Flight(user.username, flightID, departureDelay, arrivalDelay, flightTime, cancellationStatus,

          dateOfCreation, price)

  • flightID → departureDelay, arrivalDelay, flightTime, cancellationStatus, dateOfCreation, price
  • Key = user.username, flightID

Layover(user.username, layoverID, airport, timeBetweenFlights, dateOfCreation, startTime,

          endTime)

  • layoverID → airport, timeBetweenFlights, dateOfCreation, startTime, endTime
  • Key = user.username, layoverID

  1. The Software Tools

We plan on using flask for our website because that is what some of the members of our group are familiar with and because we are all most comfortable coding in Python. We do not plan on doing anything particularly fancy with the frontend so standard HTML, CSS and Javascript should be sufficient. We will use MySQL for the database because that is what we are going to learn in class so that should be helpful to us.  We will be using Github to coordinate development.

  1. The Data

We plan to get all of our data from kaggle datasets and APIs. The core of our data will be from two data sets we found, with one on flight delays and one flight pricing. We can do most of our functions with just these two, but additionally we may want to pull in live data about the weather and status of airports. This is where we will pull some data from public APIs.
